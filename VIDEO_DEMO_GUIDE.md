## Operator Page Demo Flow

### Overview Tab:
1. **System Statistics:**
   - Total users: 100
   - Total recommendations: ~600+
   - Flagged recommendations: Count of items flagged by agentic review
   - Persona distribution: Shows count of users per persona type

### User Search:
1. **Dropdown Search:**
   - Type email to filter
   - Example users appear at top with persona badges
   - Select a user to view details

### User Details:
1. **Overview Tab:**
   - User ID, consent status
   - **Persona Assignments (30d):**
     - Primary Persona: Name + Score (e.g., "high_utilization: 50%")
     - Secondary Persona: Name + Score (e.g., "variable_income: 30%")
     - **Note:** Scores don't add up to 100% - they're confidence scores, not percentages
   - Active recommendations count

2. **Signals Tab:**
   - Subscription signals: Count, monthly spend, share of total
   - Savings signals: Net inflow, growth rate, emergency fund coverage
   - Credit signals: Utilization, interest charges, overdue status
   - Income signals: Frequency, gap days, cash flow buffer

3. **Recommendations Tab:**
   - List of all recommendations (active, dismissed, completed)
   - Shows rationale, type (education/offer), agentic review status
   - Actions: View details, hide, approve (if flagged)

4. **Audit Tab:**
   - Operator actions log
   - Shows who approved/hid recommendations and when

### Flagged Recommendations:
1. **Flagged Queue:**
   - Shows recommendations flagged by agentic review
   - Displays: User email, recommendation title, rationale, flag reason
   - Actions:
     - **Approve:** Approve and show to user
     - **Hide:** Hide from user
     - **View Details:** See full recommendation details

## Common Questions & Answers

**Q: Why don't persona scores add up to 100%?**
A: Persona scores are confidence/match scores (0.0-1.0), not percentages. A user can have a 50% match to High Utilization and a 30% match to Variable Income - these are independent confidence scores, not a distribution.

**Q: How many recommendations does each user get?**
A: 3-5 educational articles + 1-3 partner offers (enforced by the system).

**Q: Are recommendations diverse?**
A: Yes! The system ensures variety across signal categories (credit, subscription, savings, income) and considers both primary and secondary personas.

**Q: What if a user matches multiple personas?**
A: They get a primary persona (highest score) and potentially a secondary persona (if score â‰¥ 30%). Recommendations consider both for diversification.

**Q: How does the system handle flagged recommendations?**
A: Agentic review flags items with tone violations or eligibility issues. Operators can approve (show to user) or hide (remove from user) flagged items.
